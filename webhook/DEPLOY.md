# Meridian CPD Webhook — Deployment Guide

## Step 1 — Supabase: Create the tables

1. Go to: https://supabase.com/dashboard/project/cpkyloywjmpesmjgqaxm/sql/new
2. Paste the contents of `supabase_migration.sql`
3. Click **Run**

---

## Step 2 — Supabase: Create the courses storage bucket

1. Supabase Dashboard → Storage → New Bucket
2. Name: `courses`
3. Public bucket: **Yes** (so PDF URLs work without auth)
4. Upload each course PDF: drag-and-drop the files from `/Users/mac/Documents/meridian-cpd/courses/`
   - The PDFs are generated by the `pdf_builder.py` script in `/courses/`
   - Filename must match: `MER-DEA-001.pdf`, `MER-DEA-002.pdf`, etc.

---

## Step 3 — Gmail: Get OAuth credentials

You need a refresh token so the webhook can send email as hello@meridiancpd.co.uk.

1. Go to: https://console.cloud.google.com/
2. Create a new project: "Meridian CPD"
3. Enable the Gmail API: APIs & Services → Enable APIs → search "Gmail API" → Enable
4. Create OAuth credentials:
   - APIs & Services → Credentials → Create Credentials → OAuth Client ID
   - Application type: **Web application**
   - Authorized redirect URIs: `https://developers.google.com/oauthplayground`
5. Note your **Client ID** and **Client Secret**
6. Get the refresh token:
   - Go to: https://developers.google.com/oauthplayground
   - Click the gear icon (⚙) → Check "Use your own OAuth credentials" → enter Client ID + Secret
   - In the left panel, find "Gmail API v1" → select `https://www.googleapis.com/auth/gmail.send`
   - Click "Authorise APIs" → sign in as hello@meridiancpd.co.uk
   - Click "Exchange authorization code for tokens"
   - Copy the **Refresh token**

---

## Step 4 — Stripe: Get Price IDs

1. Stripe Dashboard → Products
2. Click your £19.99 single course product → find the Price ID (starts `price_...`)
3. Click your £79/yr subscription product → find the Price ID
4. You'll need both for the Railway environment variables

---

## Step 5 — Railway: Deploy the webhook

1. Go to: https://railway.app → sign up (free)
2. New Project → Deploy from GitHub repo
   - Or: New Project → Empty Project → then use CLI

**Using Railway CLI (recommended):**
```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# From the webhook folder:
cd /Users/mac/Documents/meridian-cpd/webhook
railway init          # creates new project
railway up            # deploys
```

3. In Railway Dashboard → your project → Variables, add ALL the variables from `.env.example`
4. Note your Railway URL: `https://your-app.up.railway.app`

---

## Step 6 — Stripe: Register the webhook endpoint

1. Stripe Dashboard → Developers → Webhooks → Add endpoint
2. Endpoint URL: `https://your-app.up.railway.app/stripe-webhook`
3. Events to listen to: `checkout.session.completed`
4. Click Add endpoint
5. Copy the **Signing secret** (starts `whsec_...`)
6. Add it to Railway Variables as `STRIPE_WEBHOOK_SECRET`

---

## Step 7 — Test it

Send yourself a test purchase through the Stripe payment link.
Check Railway logs: Dashboard → your project → Deployments → View Logs

Expected flow:
1. Payment completes
2. Stripe fires webhook → your Railway app
3. Railway app emails the PDF + completion link to the buyer
4. Buyer clicks completion link → sees confirmation page
5. Buyer ticks the box → certificate emailed immediately
