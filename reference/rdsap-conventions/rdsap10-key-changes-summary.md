# RdSAP 10 — Key Changes Summary
**Reference document for course content and subscriber library**
**© Meridian CPD 2026. All rights reserved.**

---

## Overview

RdSAP 10 (Reduced Data Standard Assessment Procedure, version 10) was launched in
June 2025. It replaces RdSAP 9.94 and represents the most significant change to
domestic energy assessment methodology in 13 years. All new EPCs lodged after the
transition date must use RdSAP 10.

---

## 1. Windows and Glazing

**RdSAP 9.94:** Windows recorded as a proportion of floor area (e.g. "normal").
**RdSAP 10:** Windows must be measured individually by orientation (N/S/E/W) and
recorded by area in m². New glazing types added including triple glazing with
secondary data options.

**Key convention changes:**
- Measure window opening (frame to frame), not glass area
- Each elevation recorded separately
- Bay windows: measure the total area of all panes including returns
- Roof windows: recorded as horizontal or pitched, with orientation
- Secondary glazing: now a distinct option (not a proportion estimate)

---

## 2. Ventilation

**RdSAP 9.94:** Basic ventilation types — natural, extract fans, MVHR.
**RdSAP 10:** Significantly expanded ventilation options and new data requirements.

**New/changed ventilation types:**
- Passive Input Ventilation (PIV) — now a distinct category
- MVHR (Mechanical Ventilation Heat Recovery) — additional data on ducting type
- Natural ventilation: trickle vents now required where present
- Intermittent extract fans: count by room type now required
- Positive Input Ventilation (roof-mounted): distinct from wall-mounted

**Key convention:**
- Where ventilation cannot be determined by inspection, use default based on age band
- Do not assume — record only what can be evidenced

---

## 3. Room in Roof (RiR)

**RdSAP 9.94:** Single Room in Roof type.
**RdSAP 10:** Two distinct types introduced.

**Type 1 — Room in Roof (insulation at ceiling of rooms below):**
- Roof slope insulated at ceiling level
- Knee walls present or absent
- Gable walls included in assessment

**Type 2 — Room in Roof (insulation follows roof slope):**
- Insulation follows the roof slope (warm roof approach)
- More energy-efficient — affects EPC rating
- Requires evidence of insulation type and thickness

**Key convention:**
- Conversion must be evidenced (photographs + installer documentation where available)
- Default to Type 1 where type cannot be confirmed

---

## 4. Solar PV, Battery Storage, and PV Diverters

**RdSAP 9.94:** Basic PV data (kWp, orientation, pitch).
**RdSAP 10:** Expanded to include battery storage and diverters.

**New fields:**
- Battery storage: capacity (kWh), connected to PV (Y/N)
- PV diverter (immersion diverter): present (Y/N)
- Multiple PV arrays: each recorded separately with own orientation/pitch
- Community PV: new treatment for communal systems

**Key convention:**
- Battery storage significantly improves EPC rating — ensure it is not omitted
- Diverters count as reducing hot water energy — record where present

---

## 5. Airtightness

**RdSAP 9.94:** Not recorded — assumed by age band.
**RdSAP 10:** Measured airtightness can now be entered where a test result exists.

**Key convention:**
- Only enter a measured value if a valid pressure test certificate exists
- Default (no test) uses age-band assumptions
- Measured value will typically improve the EPC rating for well-sealed properties
- Acceptable test standard: ATTMA TS1 / BS EN ISO 9972

---

## 6. Heat Pumps

**RdSAP 9.94:** Heat pumps recorded as a single type.
**RdSAP 10:** Expanded data requirements for all heat pump types.

**Changes:**
- Air Source Heat Pump (ASHP): CoP source options updated
- Ground Source Heat Pump (GSHP): borehole vs horizontal loop distinction
- Hybrid heat pump systems: new treatment
- Heat pump + underfloor heating: interaction now modelled differently

**Key convention:**
- Record heat pump make and model where possible (aids CoP lookup)
- Do not estimate CoP — use default table where model unknown

---

## 7. Lighting

**RdSAP 9.94:** Proportion of low-energy lighting.
**RdSAP 10:** All lighting assumed LED/low-energy by default for properties built
after a certain date. Older properties retain percentage recording.

---

## 8. Age Bands

RdSAP 10 uses the same age band structure as RdSAP 9.94 (pre-1900 through 2023+)
but with updated default U-values and construction assumptions for each band.
The 2023+ band is new and applies to properties built to Part L 2021.

---

## 9. Default Values and Evidence

A core RdSAP 10 principle: **evidence over defaults**. Where a feature can be
measured or evidenced, the actual value must be used. Defaults are only acceptable
where the feature cannot be determined by inspection or documentation.

Auditors will increasingly challenge assessments that use defaults where evidence
was available. Photograph evidence is mandatory for non-default entries.

---

## 10. EPC Transition

Properties assessed under RdSAP 9.94 do not need to be reassessed under RdSAP 10.
However, any new lodgement (including renewal of an expired EPC) must use RdSAP 10.

Accreditation bodies issued transitional guidance allowing a short overlap period.
Check your scheme's specific transition bulletin for cut-off dates.
